What Are Web Technologies?

Building the Modern Web

Web technologies are standardised languages, protocols and tools that allow developers to create websites and web applications accessible through a browser over the internet.

The three core front-end technologies are HTML (structure), CSS (style) and JavaScript (behaviour). Together they form the foundation of every webpage.

On the server side, PHP, Python and Node.js process requests and interact with databases (MySQL, MongoDB) to serve dynamic content.

Communication happens over HTTP and HTTPS — protocols defining how browsers and servers exchange data securely.

Brief History of Web Technologies

1989–1991

Tim Berners-Lee proposes the World Wide Web at CERN and publishes the first HTML document and web server.

1993

NCSA Mosaic launches — the first graphical web browser, making the web accessible to mainstream users.

1994

W3C (World Wide Web Consortium) is founded to develop open, universal web standards.

1995

JavaScript is created by Brendan Eich at Netscape, adding scripting capability to web pages.

1996

CSS1 is released, cleanly separating content structure from visual presentation.

1997

HTML 4.0 formalises tables, forms, frames and improved multimedia support.

2004

WHATWG begins work on HTML5, creating a web application platform without plug-ins.

2014

HTML5 officially standardised — native <video>, <audio>, <canvas> and semantic elements.

2026

Progressive Web Apps, WebAssembly and AI-driven interfaces define the modern web landscape.

Modern data centres serve billions of web requests every second.


Git & GitHub underpin collaborative web development worldwide.

Web Technologies Glossary — A

HTML
HyperText Markup Language — the standard language for structuring web content using elements and tags.
CSS
Cascading Style Sheets — describes visual presentation: layout, colours, fonts and responsive behaviour.
JavaScript
A lightweight scripting language enabling dynamic, interactive content running natively in every modern browser.
HTTP
HyperText Transfer Protocol — defines how messages are formatted and transmitted between browser and server.
HTTPS
HTTP Secure — HTTP encrypted with SSL/TLS, protecting data from eavesdropping and tampering.
DNS
Domain Name System — translates domain names (e.g. www.example.com) into numeric IP addresses.

Web Technologies Glossary — B

URL
Uniform Resource Locator — the address for accessing a web resource, made of protocol, domain and path.
DOM
Document Object Model — a tree-based programming interface representing an HTML document that scripts can manipulate.
API
Application Programming Interface — rules allowing software systems to communicate, e.g. REST or GraphQL.
Responsive Design
Web design approach using fluid grids and media queries to display correctly on all screen sizes.
SEO
Search Engine Optimisation — techniques improving a website's ranking in search engine results pages.
CMS
Content Management System — software allowing non-technical users to publish web content (e.g. WordPress).

What Are HTML Frames?

⚠ Note: The <frameset> and <frame> tags were part of HTML 4 but are deprecated in HTML5. The modern alternative is <iframe>. Both are demonstrated below.

Frames allowed a browser window to be divided into independent panes, each loading a separate HTML document. A common pattern was a navigation sidebar frame + a main content frame. Links in the sidebar used target="framename" to load pages into the content frame without reloading the whole page.
